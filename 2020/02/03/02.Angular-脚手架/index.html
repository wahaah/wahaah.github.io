安装Angular-cli脚手架工具

angular一开始就走的全家桶式设计思路

Angular-cli是Angular官方开发的一个脚手架工具，专门用来构建Angular应用程序

angular应用程序初始化

安装依赖环境

1.安装node.js

2.安装npm 【npm会随着node的安装被一起安装】

3.安装python

4.安装c++编译工具

安装cnpm

安装脚手架

脚打包手架集成了webpack，开发服务器，单元测试，自动编译，部署等功能特性。

用 ng –version查看版本号

安装失败的解决方案

  • 在windows平台上安装脚手架会报很多错，是因为脚手架平台依赖python和visual studio环境，很多开发者没有安装这些东西。
  • node-sass被墙的问题，强烈推荐用cnpm安装，可非常有效的避免撞墙。
  • 若安装失败，就手动把全局的脚手架删掉，cnpm uninstall -g @angular/cli
  • 若node-modules删不掉，爆出路径过长之类的错误就强行删除

使用angular/cli初始化项目

找一个开发目录

使用该命令会创建好项目后自动装包，为避免被墙的问题，

  1. 可以使用ctrl+c停止执行
  2. cd 项目名 来切换到项目目录下
  3. 使用cnpm i去安装
  4. 安装完成后可以看到有个node_modules文件夹，该目录下第三方包是项目所有依赖项所依赖的
  5. 在项目根目录下用命令 ng serve （npm start）来启动项目，命令作用启动一个开发服务，生成一个地址帮助浏览。注：ng serve –port 4100是可以在指定端口中打开

项目的代码写在src目录下。
